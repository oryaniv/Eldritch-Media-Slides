don't be panicked, here's an explanation of what exactly i just did here:

see, one basic features of the google maps API is the creation of markers, all i did was for each object of data declared before, i create a marker to be placed on the map. when a the marker is clicked, an information window will be opened with relevant details.

what about your slideshow library??

right... well' let's.

all we have now is a nice map, with some markers, that when clicked will showing you a window with information about the marker's location. this is nice and all but isn't really... exciting. what we'll like to have, is some color - some pictures to show the magnificense of that location. oh, now we get it - a slideshow!

then here we go:


    /* the following code is within the addListener() method mentioned before */

    var slideDiv = document.querySelector("#"+slideId);
    if(slideDiv.childNodes.length > 0){
        slideDiv.removeChild(slideDiv.childNodes[0]);    
    }
    EldritchSlideShow.Create("#"+slideId,
                                {
                                Slides:e.Images,Size:1
                                ,Loop:true,CssClass:'rounded framed'
                                ,Audio: e.sound,
                                CallBack:{Function:"animate",Params:[markerIndex]}
                                }
       ).Start();
    }); 


    function animate(index){
        var marker = markers[index];
        marker.setAnimation(google.maps.Animation.DROP);
        setTimeout(function(){marker.setAnimation(google.maps.Animation.DROP);},1000);
    }

examine this code: first we take the html element with the "slideId" id as the container element for the slide. then, we make sure it is empty (if there is already a slideshow within, we'd first like to remove it before creating another). Then, we create a slideshow from the data supplied to us at the start and run it.

Note also that we use the callback propery of eldritch media slides - it is set to use the animate() method, which animets the google marker at the end of the slidshow's run.
